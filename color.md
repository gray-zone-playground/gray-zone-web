# 프로젝트 디자인 시스템 컬러 가이드 (Color Palette)

이 문서는 프로젝트의 일관된 시각적 언어와 사용자 경험을 위해 정의된 컬러 시스템입니다. 모든 UI 요소는 정의된 팔레트 내의 값을 사용하는 것을 원칙으로 합니다.

## 1. Neutral Colors (Base & Gray)

가장 빈번하게 사용되는 기초 색상입니다. 배경, 테두리, 텍스트의 계층 구조를 정의할 때 사용합니다.

| 단계         | Hex Code  | 용도 요약                              |
| :----------- | :-------- | :------------------------------------- |
| **White**    | `#FFFFFF` | 기본 배경, 밝은 텍스트                 |
| **Gray 100** | `#F5F5F5` | 보조 배경, 카드 구분선                 |
| **Gray 200** | `#E5E5E5` | 비활성화 상태 배경, 입력창 테두리      |
| **Gray 300** | `#D4D4D4` | 일반적인 Divider, Placeholder          |
| **Gray 400** | `#A3A3A3` | 보조 설명 텍스트, 비활성 아이콘        |
| **Gray 500** | `#737373` | 메인 보조 텍스트, 읽기 전용 텍스트     |
| **Gray 600** | `#525252` | 강조가 필요한 보조 텍스트              |
| **Gray 700** | `#404040` | 본문 텍스트 (Body Text)                |
| **Gray 800** | `#262626` | 제목 텍스트 (Headline)                 |
| **Gray 900** | `#171717` | 가장 강조되는 텍스트, 다크 테마 포인트 |
| **Black**    | `#000000` | 완전한 블랙 (필요 시 제한적 사용)      |

## 2. Semantic Colors (Status)

사용자에게 시스템의 상태를 직관적으로 전달하기 위한 색상입니다.

### Error (Red) - 위험, 오류, 삭제

| 단계          | Hex Code  | 용도                        |
| :------------ | :-------- | :-------------------------- |
| **Error 100** | `#FEE2E2` | 에러 메시지 배경            |
| **Error 500** | `#EF4444` | 에러 아이콘, 삭제 버튼 배경 |
| **Error 900** | `#7F1D1D` | 에러 텍스트 (심각한 오류)   |

### Success (Green) - 성공, 완료, 긍정

| 단계            | Hex Code  | 용도                   |
| :-------------- | :-------- | :--------------------- |
| **Success 100** | `#DCFCE7` | 성공 알림 배경         |
| **Success 500** | `#22C55E` | 성공 아이콘, 완료 버튼 |
| **Success 900** | `#064E3B` | 성공 메시지 텍스트     |

### Warning (Amber) - 주의, 경고, 대기 (추가)

| 단계            | Hex Code  | 용도                       |
| :-------------- | :-------- | :------------------------- |
| **Warning 100** | `#FEF3C7` | 주의 사항 배경             |
| **Warning 500** | `#F59E0B` | 경고 아이콘, 업데이트 알림 |
| **Warning 900** | `#78350F` | 경고 텍스트                |

### Info (Blue) - 정보, 안내, 링크 (추가)

| 단계         | Hex Code  | 용도                               |
| :----------- | :-------- | :--------------------------------- |
| **Info 100** | `#DBEAFE` | 일반 정보 안내 배경                |
| **Info 500** | `#3B82F6` | 정보 아이콘, 하이퍼링크, 주요 안내 |
| **Info 900** | `#1E3A8A` | 정보 텍스트                        |

---

## 3. 왜 이 컬러 시스템을 사용해야 하는가?

### 1) 상태 피드백의 직관성 (Semantic Meaning)

단순히 예쁜 색을 고르는 것이 아니라, **색상이 기능을 수행**하게 합니다. 빨간색은 '주의/에러', 초록색은 '성공'이라는 보편적인 인식을 활용하여 사용자가 텍스트를 읽기 전에도 현재 상황을 즉각적으로 파악하도록 돕습니다.

### 2) 접근성 및 명암비 (Accessibility)

각 컬러의 500번대 이상은 흰색 배경에서 충분한 대비를 가져 가독성을 보장하며, 100번대 배경색과 900번대 텍스트 색상을 조합하여 가독성 높은 알럿창(Alert UI)을 손쉽게 구성할 수 있습니다.

### 3) 디자인 확장성

Gray를 9단계로 세분화함으로써, UI 요소 간의 깊이감(Depth)을 조절할 수 있습니다. 예를 들어, 배경(100) 위에 카드(White)를 올리고, 그 위에 테두리(200)와 텍스트(700)를 배치하는 식의 정교한 설계가 가능해집니다.
